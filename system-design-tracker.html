Interview Prep · 6-Month Window

System Design // Staff Track

Mobile-weighted. Practice loop, not a reading list. Front-load the floor, then rep with answer keys.
PHASE 0Front-load — the only thing you read ahead
~1 week, 3–4 sessions. This is the entire pre-practice phase. The deeper foundations arrive inside the worked problems, not before them. Do not extend this into a reading detour.
TOPICSThe three buckets
Bucket 1 you already own from the Lloyds job — checking it off means "I recognise this as system-design vocabulary," not study. Bucket 2 is the real front-load. Bucket 3 is names-now, depth-on-demand.
PHASE 1Practice problems — recommended sequence
Vivo answer-key problems first (you can diff your design), then open-book. One rep per session, 45 min timed. Status cycles on click: Not started → Attempted → Reviewed → Solid. Reviewed = rep done + diffed. Solid = could carry a real interview.

Rep protocol

One rep = 45 min timed — Requirements 5' → HLD 10' → Deep dive 20' → Trade-offs + wrap 10'. Then diff against the Vivo chapter, or self-critique for open-book problems.

Then journal (the textarea on each problem): framework path taken · key trade-off + why · what you fumbled · the trigger that should fire next time.

Cadence: 2 sessions/week, 60–90 min each. Do not touch the 5:15–6:15 DSA slot — system design tolerates a less-fresh brain; DSA does not.
